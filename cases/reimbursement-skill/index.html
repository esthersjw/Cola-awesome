02

四步把报销变成自动的

1

把你的报销规则写下来

每个公司的报销规则不一样,但你只需要告诉AI这几件事--不需要写得很正式,就像跟一个新来的同事交代"我们报销怎么弄"一样。

  • 公司抬头和税号
  • 报销发给谁(邮箱)
  • 文件怎么命名(日期-事由-金额?)
  • 有没有特殊规则(学习津贴、外币、情况说明)
最简单:直接把公司现有的报销SOP文档复制粘贴进来就行
2

让AI写成Skill

把规则丢给Cola,说:"帮我根据这个写一个报销Skill。" 它会自动帮你整理成一套完整的流程规范--写完之后永久生效,不需要每次重新教它。

  • 什么时候触发(你说"报销"、"发票"就激活)
  • 标准流程(识别发票 → 重命名 → 归档 → 打包 → 发邮件)
  • 命名规范(按你公司的规则来)
  • 特殊情况处理(外币、替票、津贴等)
3

实战跑一遍,边跑边补

第一次用的时候直接丢一张真实发票试试。跑的过程中你可能会发现一些漏掉的规则--随时补进去就行。跑通一次之后,以后每个月就是纯自动的了。

  • "哦对,外币的还需要建子文件夹"
  • "学习津贴不用写情况说明,直接交就行"
  • "替票的命名前面加个'替票-'"
4

还没开票的,它帮你记着

不是所有费用都能马上拿到发票。以前全靠脑子记,月底一定漏。现在随口告诉Cola,它就记下来。你问"还有什么没给你的",它直接报清单。

不需要你自己维护什么表格。它记住了,你忘了也没关系。

# 报销助手

## ⚙️ 首次使用:引导用户配置

### 必填信息
1. 公司名称和税号
2. 报销发给谁(邮箱)
3. 报销人姓名
4. 文件命名规则
5. 报销文件存放位置

### 选填信息
6. 特殊津贴(学习津贴、餐补等)
7. 外币处理规则
8. 邮件格式要求
9. 报销分类
10. 其他公司特殊规则

## 核心流程

1. 接收发票 → 识别日期/事由/金额
2. 重命名   → 按配置的命名规则
3. 存放     → 当月文件夹归档
4. 待开追踪 → 主动记录、主动提醒
5. 月底打包 → 汇总 + ZIP + 确认
6. 发送邮件 → 等用户确认后发送
