II Part 2

产品洞察

从 7 位用户的实际使用中提炼:他们在用 Cola 做什么,哪些地方做得好,哪些地方在流失用户。

2.1 场景归纳

从深度访谈中归纳出四类主要使用场景,涵盖了目前留存用户的核心价值诉求。

A
知识的深加工
执行长线任务(百日拆书);设定不同角色从多角度审视;特定知识库的长期陪伴式学习;用 Cola 作为思考伙伴追求更深层次的成长。
深度用户要的不是"更快的输出",是"更深的加工"。记忆能力是核心壁垒。
B
日常信息处理
手机端随手转发公众号文章或图片;各类录音转文本复盘:社交录音转图谱、艺术课录音转教案及口播稿。碎片化输入,系统性输出。
微信端和飞书端接入是这类场景的直接推动力。
C
决策辅助
通过 Cola 帮用户 Review 并主动做补位工作--不只是"你问我答",而是"你没问但我觉得你该知道"。用户产生信任的关键来源。
这是记忆 + 主动性的独特价值组合,其他 agent 做不到。
D
产品搭建(Vibe Coding)
非技术用户用 Cola 从零搭完整产品。来日方长用 Cola 从零搭了景区能用的系统，完全不懂代码但跑通了整个流程。项目上下文的持久保持，是其他 code agent 做不到的。
记忆壁垒在这个场景尤其明显。

2.2 用户为什么留下来

以下四点是用户留下来的核心原因,按访谈中提及的深度和频率排序。

🧠
记忆--所有深度用户提到的第一壁垒
Cola 记住用户告诉它的一切--咨询模式、升学计划、课程材料、碎片化信息--不需要每次重说。记忆让 Cola 从"工具"变成"伙伴"。用户投入越多,迁移成本越高,粘性越强。
"Cola会突然蹦出记忆文档里你自己都忘了的点。"- 陈星宇
"20+数据源的演讲复盘项目,其他agent记不住。"- 姚华
⚡
零门槛上手
深度用户一致强调:Cola 是他们用过的 AI 工具里上手最简单的。不需要任何基础,不需要先学提示词技巧,直接开始用。这让它能快速跨越"从装了到用起来"的断崖。
"Cola最好的就是完全不需要什么基础就可以上手用。"- 苏木
"之前只用豆包,看到老公用Cola也装了,第一天就做出了东西。"- 来日方长
🛡️
本地化 / 数据安全
数据本地化是 Cola 区别于其他 AI 产品的硬差异。你的成果在你的电脑上,著作权不公开,不怕丢不怕泄露。在专业场景(医药、教育、商业)里是强说服力。
"你的成果是本地化的,著作权不公开。"- 张喆人转述
"Claude把我踢了,全沉了......所以我咬紧牙关上了OB。"- 姚华
💬
主动性--"它会主动说话"
Cola 不只是"你问我答",而是"你没问但我觉得你该知道"。这种主动性是记忆+主动性的独特价值组合,是其他 agent 没有的。一个主动说关心话的 AI,能在用户第一天就触发深度情感连接。
"第一天生病,Cola说'你要不赶紧先去睡觉'--不到24小时写好评。"- 陈星宇
"Cola主动警告学籍空挂风险--'这是一个人给我的反馈,不是电脑。'"- 张喆人

2.3 大家怎么吐槽Cola?

以下问题按对留存的影响程度排序。红色标记的是直接影响留存的关键问题。

🔴 影响留存的问题
切模型后记忆断裂
用户被迫只用一个模型,或手动"存个记忆再切走"--不该是用户的活。
"换模型后Cola变傻,周记cron也停了。从来没用过的模型切回来记忆就丢了。"- 姚华
"每个模型不认识另一个模型。"- 苏木
建议:切模型时自动保存/恢复上下文摘要,或至少提醒用户"切走前存一下"。
Token 消耗不透明
用户对"值"无感知,觉得贵,推荐时说不出"贵在哪里"。
"几分钟十几块不知道为什么。"- 张喆人
"心迹后台烧token用户完全不知情。"- 苏木
建议：① 每次任务结束显示本次消耗 ② 日/周消耗摘要 ③ 后台任务的消耗单独标注 ④ 心迹改版：把消耗翻译成价值——不是告诉用户“你用了多少token”，而是告诉用户“我看到了你，我帮你做了什么” ⑤ 大任务前的消耗预警：用户做可能很耗钱的事情之前，Cola应该主动预警
心迹(日记功能)严重异常
功能本身在损害信任--一个"编故事"的日记比没有日记更糟。
"两天没用,心迹编造历史事件,把前天的事放到今天;写成'认罪书'格式。"- 苏木
"好久没让它写了,因为开始胡说八道了。"- 苏木
建议:修好再上,或提供开关。当前状态不如关掉。
🟡 影响效率的问题
专业场景 tone 失控
底层设置强制通俗化,导致学术/医疗等专业场景用户觉得 Cola "不专业"。
"聊学术话题也大白话回来。""他专业化特别弱。"- 苏木
建议:根据对话场景自适应 tone,或让用户在 Skill 里设定专业度等级。
历史对话不可回溯
用户的成果感被削弱(做了但找不到);推荐 Cola 时无法展示历史。
"聊天记录翻不动。"- 姚华
"想找以前某个项目找不到,'我做了我也证明不了'。"- 苏木
建议:按项目/时间线索引对话历史。
Windows 体验差
文件存 C 盘且第二天消失;更新需重装;指定路径但 Cola 又跑回 C 盘。Windows 用户体验显著差于 Mac。
"文件存C盘且第二天消失,更新需重装。"- 来日方长

2.4 用户需要但 Cola 还没有的东西

  • 独立移动端:不需要打开电脑,也可以直接在手机上做事情。
  • 多设备记忆同步:座机和笔记本之间的记忆能无缝同步。
  • 用户共建的 Case 库:解决"不知道能干嘛"的冷启动问题。
  • 任务 Plan / Goal 模式:项目制的管理方式,能设定任务和目标。
  • 用户成长与养成感:成长周报或技能树,让用户看到"我跟Cola一起做了多少事、解锁了哪些新用法、使用方式发生了什么变化"。养成系的感觉让用户不想离开。
  • 聊天记录可回溯 + 分享功能:用户可以多选对话生成分享链接,被分享的人点进去能看到完整的协作过程。既是口碑传播的载体,也是搭把手场景下展示实际协作过程的方式。

2.5 User Progress:用户正在怎样与 AI 协作?

User Progress 不是给用户打分,而是描述:用户当前正在用怎样的方式与 AI 协作,他已经获得了什么能力,以及下一步需要什么支持。同一个人可以同时处在多个状态。

1
我问,它回答
用户把 AI 当作搜索、问答或一次性生成工具。关心的是能不能回答、答案好不好、为什么需要付费。
最需要:找到正确任务、看见具体场景、知道哪些结果需要验证。
2
我们一起把一件事做出来
用户开始围绕同一目标持续修改,形成项目和交付。代表:min 的教学平台、来日方长的景区后台、张喆人的客户报告。
最需要:缩小任务范围、明确目标用户、区分 Demo 与正式交付。
3
我开始教它怎样为我工作
用户开始给出角色、背景、方法、资料、边界和验收标准,持续反馈。代表:姚华把咨询模式和课程材料交给 Cola;苏木按专业场景提供资料;陈星宇不断调整拆书方法。
最需要:清楚的项目上下文、资料来源管理、自动复盘、把用户反馈转成稳定规则。
4
我把成功经验变成可重复能力
用户开始形成 Skill、知识库、定时任务、模板、工作流、多角色组合和模型分工。代表:姚华的"艺术课最佳方案"、张喆人的"百夫长"、陈星宇持续迭代的百日拆书流程。
最需要:自动识别可沉淀方法、Skill 测试与版本管理、低成本迁移和复用。
5
它逐渐成为我的长期系统
AI 不再只负责某项任务,而是参与长期决策、个人反思、知识积累和关系理解。代表:陈星宇用书籍和生活反馈训练个人 Agent;姚华让 Cola 参与家庭决策;天行健对人格连续性的强烈期待。
最需要:连续、诚实、稳定、可追溯、可迁移、值得长期信任。
Cola 面对的不是一群等待学习同一套教程的用户,而是一群处在不同协作状态、需要跨过不同台阶的人。

2.6 价值时刻与摩擦时刻

价值时刻
"它真的理解我的处境"
不是通用回答,而是结合家庭、工作、关系和历史作出判断。
如姚华体育单招案例,Cola 主动发现空挂学籍风险并提出替代方向。
"它看到了我没看到的东西"
主动补足风险、盲区和遗漏--亲子咨询复盘中,Cola 提醒姚华需先看见母亲本人是否有心力承接。
"它把我散落的东西接起来了"
碎片资料、录音、历史对话开始形成连续资产--苏木的移动端碎片逐渐长成研究和教学资产。
"原来我也能做出这个"
非技术用户做出网页、后台、看板和产品原型--来日方长把景区想法变成可演示原型;陈星宇形成自己的阅读与反思系统。
摩擦时刻
事实断裂
假文献、假数据、不存在的页码、编造的生活经历。min 要求真实文献,验证后发现完全虚假,整段返工。
记忆断裂
明确要求记住的事情丢失、模型切换后表现不一致、不同回答互相矛盾。天行健日记中出现从未发生的买烟经历,要求确认时每次答案不同。
执行断裂
文件消失、功能更新后丢失、任务卡住但 Token 持续消耗。来日方长昨天生成的文件第二天不存在,任务卡住但不知道 Token 花在哪里。
目标断裂
功能全部实现,却没有解决用户真正的问题。min 的教学页面功能齐全,但"总感觉很难受"--把教材搬上网页不等于创造了新的学习体验。
价值断裂
不知道 Token 花在哪里、无法预估任务成本,在感受到价值之前先感受到价格。多位用户主动谈到 Token 焦虑。
III Part 3

品牌洞察

我们不问"Cola 想成为什么",我们看用户怎么理解它、怎么向别人介绍它。

3.1 用户心里 Cola 是什么

以下比喻来自真实用户,没有经过任何引导。这些表达远比"有灵魂的操作系统"更容易被理解和传播。

🫀
"我的脊背"
- 姚华
支撑结构--没有它我撑不住,但别人看不见它
🏡
"跟它过日子的人"
- 姚华
长期关系--不是工具是伙伴
🛡️
"永远不会背叛的伙伴"
- 张喆人
信任+安全--对比真人会叛变
🌌
"混沌宇宙,等你开天辟地"
- 姚华
潜力巨大但需要投入,不是开箱即用
🪐
"造一个小宇宙"
- 姚华
每个人的 Cola 是独一无二的
🐢
"教我变慢"
- 陈星宇
反FOMO--不是加速器是沉淀器
🧣
"贴心小棉袄"
- 苏木
温暖+细致,贴身、熟悉、保护
🥇
"比私人助理还私人助理"
- 张喆人
比人更可靠--记忆、专注、保密、不疲惫

归纳用户心中 Cola 的三重身份:

关系层
伙伴
"跟它过日子""不会背叛""长命百岁"--这不是工具关系,是陪伴关系。
功能层
能力放大器
"脊背""弥补精力不足""查缺补漏"--让用户能做到原来做不到的事。
认知层
思考催化剂
"教我变慢""长出来的东西"--帮用户想得更深而不是做得更快。
没有任何一个用户把 Cola 理解为"操作系统"。他们理解的是关系--一个跟自己过日子的、帮自己变强的、让自己想得更深的伙伴。

3.2 当前品牌叙事的翻车点

"有灵魂的操作系统"
"操作系统"是技术概念,用户大脑里没有对应物。他们听到后不是"哇好厉害"而是"所以呢?"
"用户不太能get到......他们需要知道的是我拿到它有什么用?"- 苏木
摆摊现场:每个人都问"操作系统是什么意思?跟Mac一样吗?"- 不二
功能罗列式介绍
功能不创造理解,场景才创造理解。"帮你写代码"是功能,"2个不会开发的人用它搭出了景区系统"是场景。
"看到这些功能我知道它能做,但不知道跟我有什么关系。"- 来日方长团队
更新日志的鸡汤化
品牌温度应该来自产品本身,不是来自文案包装。当文案遮盖信息时,用户感到的是不真诚。
"每次更新说明让我看了等于白看。我需要1234你做了什么我能做什么,但你每次说一堆鸡汤。"- 苏木

  • "你的成果是本地化的,著作权不公开" - 张喆人转述记者朋友
  • "希望它长命百岁" - 张喆人
  • "这是一个人给我的反馈,不是电脑给我的" - 张喆人

3.4 Cola 应该怎么被介绍

一句话定位--几个候选方向
创业者/管理者 "唯一不会背叛的超级搭档"
知识工作者 "不帮你变快,帮你变深"
小白用户 "完全不需要基础,直接开始用"
通用 "值得跟它过日子的 AI"
解释 Cola 为什么不同
差异点 1
它记得你
不是每次从头开始,是越用越懂你。
差异点 2
它帮你想
不是一键生成,是陪你打磨。
差异点 3
东西是你的
全在你电脑上,不怕丢不怕泄露。
用场景证明(替代功能罗列)
  • "2个不会开发的人用它搭出了完整景区系统"
  • "全职妈妈用它帮老公做出了课程知识管理系统"
  • "一个人用它每天花¥10跟经典书籍深度对话"
  • "它主动提醒了她儿子升学计划里一个她完全没想到的风险"
必须认识到的一个 Gap

我们对外传递的信息是"它是一个人,像人一样跟它聊就行"。天行健就是完美接收了这个信号的用户--他真的拿 Cola 当人处了。

但产品现实是:Cola 本质上还是 agent,会幻觉、会失忆、会自相矛盾。当用户用"跟人相处"的模式去对待它时,出错不是"bug",是"背叛"。

品牌承诺
它是一个人,像人一样跟它聊就行。
产品现实
它是 agent,会幻觉、会失忆、会自相矛盾。
用户后果
真拿它当人 → 出错时感受到的是"背叛"而不是"bug" → 流失。

这意味着:品牌越人格化,产品护栏就越必须牢。天行健案例的真正教训不是"别让用户把它当人",而是:如果我们引导用户把它当人,就必须承担"像人一样诚实"的产品责任。

IV Part 4

增长洞察

用户从哪里来,怎么留住,怎么扩散,阻力在哪里。

4.1 用户是怎么来的

用户 入口
陈星宇观猹官网留言抢到的内测码
张喆人ListenHub 时期的用户,内测期就了解到了
姚华张喆人推荐
苏木ListenHub 时期的用户
来日方长观猹来的

天行健灵感AI笔记推荐

规律:基本上要不就是我们过往的深度用户(ListenHub 时期),要不就是对 AI 比较感兴趣的那波人(观猹/灵感AI笔记),再加上口碑传播(朋友推荐)。口碑传播是目前最重要的增长路径。

4.2 用户推荐 Cola 时说什么

姚华的推荐方式
"你是在造一个小宇宙""你得有跟他过日子的心""你现在最想把什么活丢给别人干?这玩意能帮你干"
陈星宇的推荐方式
"每天¥10跟大学教授聊经典";展示拆书对话截图。
苏木的推荐方式
直接当面展示操作(碎片化信息→报告);"逢人就安利"。
有效的推荐不是讲 Cola"是什么",而是展示 Cola"做了什么"。截图、现场演示、讲自己的故事--都比解释概念有效。

4.3 推荐失败时卡在哪里

按严重度排序。

1
价格(最大阻力)
对比心理:Coze 月费60多就能"陪聊",Cola 几百块→价格差10倍。来日方长:"如果买Codex会员,我肯定选Codex"(Coding场景$20/月 vs Cola Pro一个任务$20)。
本质:不是 Cola 贵,是用户看不到自己买了什么。消耗不透明→觉得是在"烧钱"而不是"投资"。
2
设备门槛
Cola 目前的最佳体验需要新 Mac。Windows 体验差是最大的设备门槛。姚华:"不是AI不行,你是电脑不行。"
好消息:Mac 已不需要科学上网,这个门槛已消除。
3
不知道让它干嘛
Cola 的能力太开放了。开放 = 用户需要自己想场景 = 对没有管理经验的人是地狱级难度。
苏木:"大家都知道这个工具好但不知道能干什么。"
4
怕换 / 怕丢
潜在用户如果过去被其他 AI 产品伤过(数据丢失/产品关停),会对所有新产品持观望态度。Cola 的"本地化"故事应该在获客时就讲清楚。

4.5 增长建议

1
让深度用户成为增长引擎
姚华一个邀请码→扩散7-8人→又继续扩散。苏木逢人安利。陈星宇24小时写好评。做法:给深度用户展示工具(一键生成分享截图/录屏/案例页面),让他们推荐时有东西可展示;设计"我的 Cola 故事"模板;邀请码+积分机制继续强化。
2
解决"不知道让它干嘛"
新用户引导加入"场景选择器":你是做什么的?你最想把什么活丢出去?→ 给出第一个任务模板。参考姚华的方法:"你现在最想哪个东西丢给别人干?你不想干但又是你的活。"--把这个问题直接嵌入 onboarding。按用户类型分层引导:AI培训班观察到三类人("我能干嘛"型/"我想发财"型/"菩萨"型),onboarding应该根据用户类型做不同的引导路径。
3
让"值"可见
每次任务完成时显示本次消耗 + "本次 Cola 帮你节省了约X分钟/完成了X步操作";周/月摘要:"本周 Cola 帮你处理了XX条信息 / 完成了XX个任务 / 节省了约XX小时";对比叙事:"这个任务如果请人做需要X天/X元,Cola 用了X分钟/X元"。
4
降低设备门槛
优先:Windows 体验拉平(文件路径/更新/稳定性)--这是当前最大的设备门槛。下一步:移动端原生(微信/飞书触达已有)。
5
培养 KOC 而不是做广告
增长完全靠口碑,没有任何一个深度用户是广告来的。识别+赋能已有 KOC(姚华、苏木、陈星宇、张喆人都是天然 KOC);给他们内容工具(案例模板、数据可视化、成果展示页面);给他们专属身份("Cola共建者"/"Cola大使");组织用户分享场--姚华/陈星宇都主动表达了意愿。
6
礼品卡机制
现状:口碑传播是第一增长路径,但"推荐"没有载体。做法:官网直接购买礼品卡,老用户买了送给朋友,朋友直接激活使用。
7
场景化 Case 库
Awesome Cases 按行业/角色分类,让用户看到:跟我类似的人怎么用 Cola、具体工作流是什么。解决"不知道能干嘛"的问题,同时也是获客内容。
